= opsi Extensions

Even if opsi is open source, there are some additional components that were created or maintained as part of a co-funding project and are not (yet) free of charge.

At the moment (December 2016) these are:

* MySQL backend for configuration data (see xref:server/opsi-backends#opsi-manual-backends-mysql-configdata-overview[mysql backend for configuration data])
* UEFI support (see xref:modules/uefi#opsi-manual-uefi[opsi with UEFI])
* opsi license management module (see xref:modules/licensemanagement#opsi-manual-licensemanagement[opsi-Lizenzmanagement])
* Support for clients over WAN/VPN (see xref:modules/wan-support#opsi-manual-wansupport[opsi WAN/VPN extension])
* opsi WIM Capture (see xref:modules/wim-capture#opsi-manual-wimcap[opsi WIM Capture])
* opsi local image (see xref:modules/local-image#opsi-manual-localimage[opsi local image]) and opsi-vhd-reset (see xref:modules/vhd#opsi-manual-vhd[opsi vhd reset])
* opsi Linux agent (see xref:modules/linux#opsi-manual-linux[opsi Linux Support])
* opsi Nagios connector (see xref:modules/nagios-connector#opsi-Nagios-Connector[opsi Nagios Connector])
* User roles (see xref:configed#opsi-manual-configed-hostproperties-userroles[Management of user rights and roles])
* the Scalability1 extension to increase performance in very large installations

You can find more information on this topic at link:https://www.uib.de/en/opsi-cofunding/cofunding/[opsi cofunding projects].

As long as the additional extensions are in the cofunding state, they can only be used freely for evaluation purposes, but for permanent
and normal use the cofunding contributions must be paid.

The modules are unlocked by license files.
These files have the extension `.opsilic` and are stored in the directory `/etc/opsi/licenses` on the opsi configserver.
The license files can be placed manually in this directory or uploaded via the opsiconfd admin interface (https://<configserver>:4447/admin/#licensing). This page also displays information about the current licensing.

In the past, modules were unlocked via the `/etc/opsi/modules` file.
This file is still valid.
However, new licenses are only issued in the new format.

If no license files are available, only the free components of opsi are active.

To obtain a temporary license for evaluation purposes, please contact info@uib.de.
In the context of a participation in the corresponding co-financing projects you will receive a license file for the regular use of the unlocked components.

Check the activation using one of the following methods:

In the opsiconfd admin interface (https://<configserver>:4447/admin/#licensing), licensing information is displayed.

In the opsi-configed you can display the status of your activation via the menu item 'Help / Installed opsi modules'.

image::modules-status.png["Display of the activation in {opsi-configed}",width=200]

You can use the `backend_getLicensingInfo` method to check which modules are unlocked with {opsi-admin}.


== Exceeding the licensing
Before exceeding the number of licensed clients an appropriate warning appears in the opsi-configed.
The warning is controlled by a percentage and an absolute limit.

The default values are:

* Percentage: Warning when the client count reaches 95% of the licensed client count.
* Absolute: Warning when the absolute number of free client licenses falls to or below 5 clients.

The limits can be controlled via the config parameters `licensing.client_limit_warning_percent` and `licensing.client_limit_warning_absolute`.

These values can be set via `opsi-admin` for example:
[source,prompt]
----
# Warning when 90% of licensed clients are reached:
opsi-admin -d method config_createUnicode licensing.client_limit_warning_percent "licensing - client limit warning percent" 90 90
# warning if there are only 10 free client licenses left:
opsi-admin -d method config_createUnicode licensing.client_limit_warning_absolute "licensing - client limit warning absolute" 10 10
----

If the licensed client count of a module is exceeded, the corresponding module will be disabled.
However, other modules remain active.
