= Grafana

The Grafana web application visualizes data from different data sources and displays them in so-called dashboards in the browser. Since opsi 4.2, the Grafana server displays performance metrics of the `opsiconfd` service. You open the Grafana dashboard via the xref:server:components/opsiconfd.adoc#server-components-opsiconfd-admin-page[Admin page], _Grafana_ tab. As soon as you click on the tab, the *opsiconfd main dashboard* is created or updated on the Grafana server. In addition, the user `opsidashboard` is created, which is used to access the dashboard.

The default setting is _Last 5 minutes_; however, you can customize the display by clicking on this field in the top right-hand corner. In the menu that opens, define either an absolute time period (fields _From_ and _To_) or a relative time period starting from the current time (_Last 5 minutes_, _Last 15 minutes_ etc.). Finally, click on _Apply time range_.

NOTE: You do not need Grafana on opsi-Depotservers. It is best to install `opsi-server` or `opsi-server-expert` on such servers instead of the `opsi-server-full` package. Both packages do not list Grafana in the dependencies.

[[server-components-grafana-config]]
== Configuration

The `opsiconfd` service accesses the central Grafana instance via the URL `\http://<api-key>@grafana:3000`. This is defined in the file `/etc/opsi/opsiconfd.conf`:

[source,toml]
----
grafana-internal-url = http://opsiconfd:SbihIuvo@localhost:3000
----

To connect an external Grafana instance, define it behind the parameter `--grafana-external-url`. As the last listing shows, you can enter the access data for Grafana in the URL.

TIP: After changing the `opsiconfd` configuration, you can reload it using the command `sudo opsiconfd reload`. Further information on configuring the service can be found in chapter xref:server:components/opsiconfd.adoc[The *opsiconfd* service].
