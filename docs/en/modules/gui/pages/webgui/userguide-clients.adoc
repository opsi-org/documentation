= opsi-Clients
All clients for the selected servers are listed on the Clients page. 
To view and manage clients, navigate to *Clients > All Clients*.

image::webgui/opsi-webgui_clients.png[Clients]

[[opsi-manual-opsiwebgui-opsiclients-creation]]
== Create a client
Creating a client and adding it to an {opsi-server} are the first steps to manage them. 
To create a {opsi-client}, +

. Navigate to *Clients > Creation*.
. Fill out the form. +
image:webgui/opsi-webgui_clientcreation.png[Client Creation, width=600]
. Click on *[ Create ]*.


NOTE: Client creation requires atleast the *ID* field. 
The created client will be added to the {opsi-configserver}.

[[opsi-manual-opsiwebgui-opsiclients-actions]]
== Client Actions
There are multiple actions that can be performed for a specific client.
To select a client action:

. Navigate to *Client > All Clients*.
. Select the client you want to perform action on.
. Click onimage:webgui/opsi-webgui_buttonclientactions.png[Actions]or right-click the table row. 
. Click on the required action. 
It will show a popup with confirmation.
. Click on *[ Delete ]* for client deletion and *[ Confirm ]* for other actions. +
image:webgui/opsi-webgui_clientactions.png[Client Actions, width=200]


[[opsi-manual-opsiwebgui-opsiclients-actions-clientagent]]
=== Deploy Client Agent image:webgui/opsi-webgui_buttondeployagent.png[Deploy Client Agent]
In order to connect a client to OPSI and register it as {opsi-client}, the {opsi-client-agent} must be installed on it.
The deployment of the {opsi-client-agent} is also possible via *Clients > Creation*. +

NOTE: The deployment of the {opsi-client-agent} is not required if operating system is installed with OPSI.

image::webgui/opsi-webgui_clientdeployagent.png[Deploy Client Agent, width=300]

[[opsi-manual-opsiwebgui-opsiclients-actions-ondemand]]
=== OnDemand image:webgui/opsi-webgui_buttonondemand.png[OnDemand]
You can push installations and apply changes immediately to the client by executing `on_demand` event on selected clients.  +

NOTE: It is possible to execute `on_demand` on multiple clients. 
Select clients and click image:webgui/opsi-webgui_iconondemand.png[Ondemand Icon] in the Quick Panel. 
It shows a confirmation popup with list of selected clients. 
A confirmation popup will appear with an editable list of selected clients.

image::webgui/opsi-webgui_clientondemand.png[On Demand, width=300]

[[opsi-manual-opsiwebgui-opsiclients-actions-notify]]
=== Notify image:webgui/opsi-webgui_buttonnotify.png[Notify]
Notifying a client is useful in situations that require the attention of the client's user.
This action can be used to send an alert or message to the selected clients before forcing a reboot or other immediate actions.

image::webgui/opsi-webgui_clientnotify.png[NotifyClient, width=300]

[[opsi-manual-opsiwebgui-opsiclients-actions-reboot]]
=== Reboot image:webgui/opsi-webgui_buttonreboot.png[Reboot]
This action immediately executes the reboot event on the selected client.

image::webgui/opsi-webgui_clientreboot.png[ClientReboot, width=300]

[[opsi-manual-opsiwebgui-opsiclients-actions-delete]]
=== Delete image:webgui/opsi-webgui_buttondeletion.png[Delete]
Deleting a client removes it and all associated data from the database.

image::webgui/opsi-webgui_clientdeletion.png[Client Deletion, width=300]

[[opsi-manual-opsiwebgui-opsiclients-groups]]
== Client Groups
lients are mainly grouped into `groups`, `clientdirectory` and `clientlist`.
A client can appear in the `clientdirectory` only once, but can appear in the `groups` multiple times.
A client that is not a member of the `clientdirectory` is always assigned to its `NOT_ASSIGNED` subgroup. 
All clients of the selected depots are listed in `clientlist`.

In the Quickpanel, the client groups are displayed in a tree structure under the tab image:webgui/opsi-webgui_quickpanel_tab_clients.png[Clients tab in the Quickpanel].
The number in the icon indicates the number of selected clients.
The first tab image:webgui/opsi-webgui_quickpanel_tab_selection.png[Selection tab in Quick Panel] shows a quick view of the editable list of all selections.

image::webgui/opsi-webgui_quickpanel_tab_clients_content.png[Clients tab content in quickpanel, width=300]
// image::webgui/opsi-webgui_clientgroups.png[Client Groups]

TIP: After the group selection, sort the clients table by `Selection` to get a detailed view.

[[opsi-manual-opsiwebgui-opsiclients-attributes-parameter]]
== Client Attributes and Parameters
There are two ways to edit the client attributes and parameters: Either via the *Clients > Configuration* page, or *Clients > All clients* and the configuration buttonimage:webgui/opsi-webgui_buttonconfig.png[configuration]. The latter has the advantage that you can switch between the clients more quickly, as the configuration is displayed next to the clients.

image::webgui/opsi-webgui_clientconfig.png[Client Configuration, width=600]

[[opsi-manual-opsiwebgui-opsiclients-logs]]
== Client Logs
To view client logs,

. Navigate to *Clients > All Clients*. 
. Click image:webgui/opsi-webgui_buttonlog.png[Logs Button] in the table row. 
Or right-click on the table row and select *Logs* from the context menu.
. Choose a log type from the list of `instlog`, `clientconnect`, `userlogin`, `bootimage`, `opsiconfd`.
. Enter a filter in the filter field or select the log level to filter logs.

By default, the log type is `instlog` and the log level is 5. 
Logs can be also viewed in *Clients > Logs*.

image::webgui/opsi-webgui_clientlogs.png[Client Logs, width=800]
