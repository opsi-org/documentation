[[opsi-manual-opsiwebgui-userinterface]]
= Benutzeroberfläche
Dieser Abschnitt soll dabei helfen, sich mit der Benutzeroberfläche vertraut zu machen.
Die {opsi-webgui} ist sowohl für Mobilgeräte als auch Desktopgeräte abgestimmt. Darüber hinaus wurde auf Barrierefreiheit geachtet.
Damit jeder Benutzer von nahezu jedem Gerät die webgui verwenden kann.

[[opsi-manual-opsiwebgui-userinterface-mainlayout]]
== Hauptlayout 

image::opsi-webgui_mainlayout.png[MainLayout]

Die folgende Liste beschreibt die Hauptkomponenten der Benutzeroberfläche:

[horizontal]
Menü:: die obere Leiste image:opsi-webgui_breadcrumb.png[Breadcrumb] enthält eine Breadcrumb-Navigation, die Benutzern ihren Standort innerhalb der Anwendung anzeigt.
Das Menü enthält auch Optionen wie
xref:opsi-manual-opsiwebgui-tips-selections[Auswahl zeigen],
xref:opsi-manual-opsiwebgui-opsiclients-actions-ondemand[On Demand],
xref:opsi-manual-opsiwebgui-settings-language[Sprache],
xref:opsi-manual-opsiwebgui-settings-theme[Theme],
image:opsi-webgui_buttonlogout.png[Logout]Logout, sowie
xref:opsi-manual-opsiwebgui-settings-quicksave-disable[Änderungen verfolgen] wenn die Sofortspeicherung deaktiviert ist.

Navigationsleiste:: das linke Menu wird verwendet, um zu verschiedenen Seiten zu navigieren.
Es zeigt auch die verbleibende Session Zeit nach der man automatisch abgemeldet wird.
Die Leiste kann in der Desktop-Ansicht mittels *<<* eingeklappt/verkleinert werden.
Die folgende Liste beschreibt verschiedenen Seiten der Anwendung: +
* Server
  - Alle Server: Verwaltung der opsi-Server (Config- und Depotserver).
  - Konfiguration: Bearbeitung der Attribute und Parameter eines opsi-Servers.
* Clients
  - Alle Clients: Verwaltung der {opsi-clients}.
  - Erstellung: Erstellung eines neuen {opsi-client}.
  - Konfiguration: Bearbeitung der Attribute und Parameter eines opsi-Clients.
  - Logs: Anzeige der Logs eines {opsi-clients}.
* Produkte: Produkte bereitstellen, Abhängigkeiten überprüfen oder Eigenschaften bearbeiten.
* Support: Anzeige der opsi-Dokumentation sowie weiterführunde Links.
* Einstellungen: Verwaltung allgemeiner oder OPSI-Einstellungen.

Seiteninhalt:: Enthält seitenspezifischen Inhalt.

[[opsi-manual-opsiwebgui-userinterface-pagecontent]]
== Seiteninhalt-Layout
Dieser Abschnitt erläutert den typischen Bereich innerhalb der Seite.
Beispielsweise zeigt die Seite *Clients* die Liste der Clients und client-spezifische Optionen.

[horizontal]
Kopfzeile:: befindet sich oberhalb der Tabelle ist eine ein-/ausklappbare Leiste, die weitere Optionen enthält. Dazu zählen beispielsweise eine xref:opsi-manual-opsiwebgui-opsiservers-treeselect[Server] Liste und xref:opsi-manual-opsiwebgui-opsiclients-groups[Client Gruppen] Auswahl, welche das Umschalten zwischen Servern und Clients ermöglichen.
Tabelle:: zeigt eine Liste von Elementen, die verzögert nachgeladen werden, wenn man nach unten (bzw. oben) scrollt.
Sie enthält diverse Möglichkeiten um die Tabelle zu bedienen, wie +
* image:opsi-webgui_buttonclearselect.png[ClearSelection] um die Liste aktuell ausgewählter Elemente zu leeren.
* image:opsi-webgui_table_filter.png[Filter] um die Tabelleninhalte nach der ID zu filtern.
* image:opsi-webgui_table_columnselect.png[ColumnSelection]um Spalten ein- oder auszublenden.
* image:opsi-webgui_table_sort.png[Sortieren]um die Tabelle nach einer Spalte auf- oder absteigend zu sortieren.
* image:opsi-webgui_table_refresh.png[Refresh] um den Tabelleninhalt zu aktualisieren. +

Die Tabellenzeile enthält Element-spezifische Optionen wieimage:opsi-webgui_buttonconfig.png[Config]Konfiguration oderimage:opsi-webgui_buttonlog.png[Logs]Logs, die eine geteilte Ansicht für die gewählte Option öffnet, oder einimage:opsi-webgui_buttonclientactions.png[Aktionen]Aktionsmenü.
Diese tabellen- und element-spezifischen Optionen können auch durch das Kontextmenü bedient werden (Rechtsklick auf die Tabellenzeile). +
Fußzeile:: Unterhalb der Tabelle befindet sich eine Möglichkeit zur (manuellen) Tabellenseiten-Navigation, was auch über scrollen erreicht werden kann.

NOTE: Das Layout für Mobilgeräte weicht leicht von dem der Desktops ab.
Beispielsweise gibt es keine geteilte Ansicht für die Seiten (Konfiguration, Logs, ..), sondern eine überlappende Ansicht.
